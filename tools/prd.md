产品需求文档 (PRD) - 分层状态管理系统 (V12 版)
1. 产品定位
一套基于心理状态分层与场景化管理的日程执行系统。它不仅管理“做什么”，更管理用户在不同任务下的“心理切换”与“执行深度”。
2. 已实现功能模块 (V12)
2.1 任务分层管理 (The Layers)
系统固定分为四个维度，每个维度具备独立的管理逻辑：
专注 (Focus)：采用“艾森豪威尔四象限”逻辑，强制任务按优先级 (P1-P4) 排序。
强健 (Sturdy)：采用“动作清单”逻辑，弱化优先级，强调大号 Checkbox 的打卡感。
缓冲 (Buffer)：采用“能量菜单”逻辑，提供不同能耗（低、中、高）的休息方案。
杂物 (Admin)：采用“极简批处理”逻辑，紧凑展示，快速清空琐碎事务。
2.2 极速交互引擎 (The Input)
一行式极速录入：任务名称与执行分钟数（Minutes）在同一行展示，支持 Tab 切换时的默认值自动注入。
场景化选项：针对不同模式展示特定表单（如：专注模式下的优先级下拉框、缓冲模式下的能耗单选框）。
2.3 巨幕执行模式 (Signboard Mode)
全屏告示态：执行任务时进入沉浸式全屏，大字号展示“对外留言”与“倒计时”。
执行闭环：支持“暂告段落（保留任务）”与“目标达成（销毁任务）”两种结束逻辑。
高保真 UI：具备精致的 SVG 播放按钮及动态交互反馈。
2.4 数据审计与日志 (The Logs)
全动作埋点：自动记录 添加、删除、开始、挂起、完成 等所有操作。
AI 友好格式：日志以 [时间] [模式] [动作] | 任务名 (时长) 格式存储，支持一键复制，方便投喂给大模型进行每日复盘分析。
3. 待优化需求 (未来规划)
3.1 智能能量电池 (Energy Battery) —— 核心预研方向
动态水位计：在页面顶部增加可视化的能量条。
非对称充耗算法：
损耗：按 时长 × 模式基准系数 扣除电量。
回充：按 Buffer 时长 × 充能系数 恢复电量。
自然衰减：引入“腺苷累积”模型，随清醒时间自然损耗电量。
3.2 任务强度系数 (Task Intensity)
个性化损耗：在录入任务时增加“强度滑块”（0.5x - 3.0x），解决“同样是专注，面试比写代码更累”的痛点。
3.3 闭环体感校准 (Feedback Calibration)
主动同步 (Sync)：用户点击能量百分比，手动修正当前体感电量。
基线自我演进：程序根据“理论计算值”与“用户体感值”的差额，自动微调（而不由用户手动调节）各模式的损耗/充能基线系数。
4. 技术栈与存储
技术：HTML5, Tailwind CSS, JavaScript (Vanilla JS)。
存储：浏览器本地存储 (LocalStorage)。
任务：os_tasks_v12
日志：os_logs_v12
设置：os_settings_v12
5. 当前版本说明
目前系统以 V12 代码为准，其特点是交互最快、逻辑最稳、功能最纯粹。它完美解决了任务的分层分类、自定义时长、以及执行时的告示显示需求，是目前最适合日常高频使用的版本。
